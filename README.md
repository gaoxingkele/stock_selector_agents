# A股多智能体选股系统

**Dual-Pipeline Stock Selector: LLM Multi-Agent Debate + Quantitative Technical Analysis**

双管线架构 — **链路A**（LLM多智能体辩论）+ **链路B**（量化技术过滤），Borda Count跨模型融合，自动生成PDF投研报告。

---

## 整体策略

系统采用**双管线互补**架构，从两个维度筛选短线强势股：

- **链路A（LLM驱动）**：先选板块再选股，6位风格互补的AI专家深度辩论，侧重**逻辑推理与多维验证**
- **链路B（量化驱动）**：全市场扫描，逐层量化过滤（量价→缠论→看图），侧重**技术信号与形态确认**

两条管线**殊途同归**：最终都汇入 Borda融合 → 风控审查 → PDF报告。可单独运行，也可同时运行取交集/并集。

| 维度 | 链路A | 链路B |
|------|-------|-------|
| **方法论** | LLM多智能体辩论 | 量化指标逐层过滤 |
| **覆盖范围** | 板块内选股（~50只） | 全市场扫描（~5000只） |
| **LLM占比** | 全程深度（E1-E7分析） | 仅L3看图投票 |
| **数据依赖** | 基本面 + 技术面 + 事件面 | 纯技术面（K线/缠论/波浪） |
| **速度** | 慢（N模型×7专家） | 快（L1/L2纯计算，L3看图） |
| **适用场景** | 深度研究，中短线 | 快速扫描，短线 |

## 智能体说明

### 市场环境层

| 代号 | 名称 | 所属链路 | 职责 |
|------|------|---------|------|
| **MR** | 市场雷达 (MarketRadar) | A | 全面板投票，扫描大盘趋势+情绪温度+概念炒作预判+ETF活跃信号 |
| **MR2** | 精简市场雷达 (MarketRadarLite) | B | 轻量版市场判断，单模型快速输出趋势+情绪+仓位建议 |

### 板块选择层（仅链路A）

| 代号 | 名称 | 职责 |
|------|------|------|
| **SP** | 板块优选专家 (SectorPicker) | 全面板多模型投票，选出 Top 3-5 板块 |
| **GX** | 跨市场主题专家 (GlobalThemeAdvisor) | 参考美股/港股热点，补充A股联动板块（最多+1个） |
| **S0** | 板块初筛 (SectorScreener) | SP降级备用：3模型快速投票 |

### 个股分析层 — 链路A（每模型内顺序执行，共享会话历史）

| 代号 | 名称 | 风格 | 持有期 | 核心分析维度 |
|------|------|------|--------|-------------|
| **E1** | 动量换手型专家 | 情绪信号 | 1-10日 | 量价动量、换手率、涨停板效应、短期超跌反弹 |
| **E2** | 成长估值型专家 | 排雷 | 3-8日 | 技术启动确认 + 基本面无硬伤 + 资金共振 |
| **E3** | 多因子平衡型专家 | 动量为王 | 3-8日 | 动量因子 + 量价共振 + 基本面排雷综合评分 |
| **E4** | 技术形态型专家 | 核心 | 1-15日 | K线形态、MACD/RSI/布林带共振，**支持K线图视觉分析** |
| **E5** | 资金流向型专家 | 跟庄 | 3-20日 | 主力净流入、大单买卖、融资余额变化、北向资金 |
| **E6** | 事件催化型专家 | 事件驱动 | 1-15日 | 政策催化、公告事件、行业景气度拐点 |

> 短线3-8天优化权重：E4技术(30%) > E3多因子(25%) > E5资金(15%) > E1动量(10%) = E2成长(10%) = E6事件(10%)

### 个股分析层 — 链路B（量化逐层过滤）

| 层级 | 名称 | 输入→输出 | 方法 |
|------|------|----------|------|
| **L1** | 量化过滤 | ~5000只→50-200只 | 排除利空形态 + 筛入量能放大/价格动量 |
| **L2** | 形态过滤 | 50-200只→10-30只 | 缠论买点信号 + 波浪量价齐升 |
| **L3** | 看图判定 | 10-30只→Top10 | K线图生成 + 多LLM模型看图投票 |

### 看图分析（链路A/B共用）

| 代号 | 名称 | 职责 |
|------|------|------|
| **CA** | 图表分析师 (ChartAnalyst) | 多模型并行分析K线图，输出买入/卖出/观望信号 |

### 独立推荐线（仅链路A）

| 代号 | 名称 | 职责 |
|------|------|------|
| **EA** | 事件驱动分析师 (EventAnalyst) | 新闻→因果链→受益标的推理，全面板投票 |
| **BA** | 异动爆发分析师 (BreakoutAnalyst) | 涨停/放量突破股扫描→题材共振评分 |

### 裁决与风控层（共用）

| 代号 | 名称 | 职责 |
|------|------|------|
| **V** | 投票仲裁者 (VotingArbitrator) | 加权投票聚合，计算共识度 |
| **R** | 风控审查 (RiskController) | ST/涨停/退市排除，三视角辩论，仓位建议 |

### 融合算法

**Borda Count**：每个模型对各股排名，第1名得20分，第2名得19分，……第20名得1分，第21名以后得0分。各模型得分汇总后排序，输出 Top 25，并附带：
- `recommended_by`：推荐该股的所有模型及其排名/分数
- `borda_score`：总 Borda 分
- `model_count`：推荐模型数量
- `all_reasonings`：各模型给出的核心理由（最多3条）

### Cloubic 路由

[Cloubic](https://cloubic.com) 是 AI 多模型统一调度平台，一个 API Key 即可访问 100+ 大模型。

- `--cloubic`：强制启用 Cloubic 路由
- `--direct`：强制全部直连（禁用 Cloubic）
- 白名单配置：`.env.cloubic` 中 `CLOUBIC_ROUTED_PROVIDERS`
- kimi 始终直连（代码硬编码），不走 Cloubic

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 配置 `.env`

```bash
cp .env.example .env   # 或直接编辑 .env
```

必填项：
```ini
TUSHARE_TOKEN=你的token
LLM_PROXY=http://127.0.0.1:7890     # LLM代理（可选，grok/perplexity需要）
GROK_API_KEY=xai-xxx                 # 至少一个LLM的API Key
```

Cloubic 模式（推荐）：
```ini
# .env.cloubic
CLOUBIC_ENABLED=true
CLOUBIC_API_KEY=sk-xxx               # 一个Key访问openai/claude/gemini/glm
CLOUBIC_ROUTED_PROVIDERS=openai,gemini,claude,glm
```

### 3. 运行

```bash
# ── 链路A（默认）：LLM多智能体辩论 ──────────────────────────
python main.py                                    # 全自动（自动选板块）
python main.py --sectors 人工智能,机器人,半导体    # 指定板块
python main.py --stocks 000988,002506,000545       # 指定个股（跳过板块）
python main.py --models minimax,doubao,openai      # 指定模型
python main.py --strategist                        # 启用首席策略师

# ── 链路B：量化技术过滤 ─────────────────────────────────────
python main.py --link b                            # 全市场扫描 L1→L2→L3
python main.py --link b --no-confirm               # 跳过L2后的手动确认

# ── 双管线同时运行 ──────────────────────────────────────────
python main.py --link both                         # 先跑A再跑B

# ── 链路B独立脚本 ───────────────────────────────────────────
python run_l1.py                                   # 仅运行L1量化过滤
python run_b_from_l2.py                            # 从L2结果继续（复用已有L2数据）

# ── 通用选项 ────────────────────────────────────────────────
python main.py --cloubic                           # 强制 Cloubic 路由
python main.py --direct                            # 强制全部直连
python main.py --demo                              # 演示模式
python main.py --config                            # 查看当前配置
```

## 技术特性

### 链路A
- **ConversationSession 上下文累积**：E1 发送完整个股画像（~53k tokens），E2-E6 仅发指令，共享历史上下文，避免重复传输
- **模型纵向并行**：每个 LLM 独立运行完整 E1→E6 流程，互不干扰，最后 Borda 融合
- **E1-E7 二次辩论验证**：Borda Top15 再经一轮完整专家辩论，补充多空风险提示
- **事件驱动独立线**：新闻因果链推理 + 异动爆发扫描，与主推荐线互补

### 链路B
- **L1 全市场量化扫描**：排除三连阴/阴包阳/破位等利空形态，筛入量价异动信号
- **L2 缠论+波浪双重确认**：分型/笔段/买点识别 + Elliott波浪量价齐升信号
- **L3 多模型看图投票**：K线图+指标图自动生成，多LLM并行看图，Borda融合共识
- **独立脚本**：`run_l1.py` / `run_b_from_l2.py` 支持分步运行和断点续接

### 共用基础设施
- **Preflight 连通性预检**：启动前并行测试所有数据源和LLM模型的连通性
- **DNS 公共解析修复**：代理软件劫持国内域名时，用 114.114.114.114 解析真实 IP
- **Cloubic 透明路由**：按白名单决定每个 provider 走 Cloubic 还是直连，运行时动态切换
- **视觉分析路由**：`supports_vision=true` → 主模型直接处理图片；`vision_model` 非空 → 路由到专用视觉模型；否则降级为文字描述
- **三级数据降级**：TDX本地（最快）→ AKShare → Tushare，任一失败自动降级
- **三视角风控辩论**：保守/中性/激进三个视角独立审查，综合裁决
- **记忆学习系统**：跨运行积累决策经验，匹配相似市场环境注入参考
- **全 try/except 保护**：新数据字段获取失败不影响主流程
- **Queue-based 异步日志**：写文件在独立后台线程，不阻塞分析流程
- **并行进度监控**：后台线程每30秒打印各模型进度快照
