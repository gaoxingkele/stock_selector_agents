## P1：推荐质量提升

### 5. A/B 管线交叉验证

- 当前两条管线完全独立运行
- 改进：A 和 B 都推荐的股票信心最高（交集加分），只有一条推荐的降权
- 链路B的L1作为链路A的**硬约束**——A推荐但B的L1都没通过的股票，应标记警告或降权
- 输出时标注"双管线共识"vs"单管线推荐"

### 6. LLM 幻觉校验

- LLM可能编造政策利好、虚构PE/PB数值、随机解读K线图
- 专家输出的关键数值（PE、涨幅、资金流向方向）与 data_engine 真实数据交叉校验
- 偏差超过阈值（如PE偏差>30%）时标记"数据存疑"，降低该分析的权重

### 7. 组合层面风控

当前风控只看单只股票，缺少组合维度：

- 同板块推荐不超过 2 只（避免集中风险）
- 总仓位根据 MR 情绪温度动态调节（强势满仓 / 弱势半仓 / 极弱空仓）
- 推荐列表的整体 beta 和相关性检查

### 8. 市场择时硬约束

- MR/MR2 判断为"弱势/下跌"时，自动缩减推荐数量
- 大盘破位（如跌破60日均线+量能放大）时，输出"观望为主，不推荐"
- 当前只是"建议"，应变成流程中的硬性门槛

## P2：持续增强

### 9. 链路B量化层增强

| 层级 | 当前逻辑 | 增强方向 |
|------|---------|---------|
| L1 | 排除利空形态 + 量能放大 | +RPS相对强度排名、行业动量排名、筹码集中度 |
| L2 | 缠论买点 + 波浪信号 | +均线多头排列打分、MACD底背离、突破前高 |
| 整体 | L1→L2串行过滤 | 改为并行打分（L1量价分 + L2形态分 = 综合技术评分），避免好股被单一条件误杀 |

### 10. 数据源扩充

| 缺失数据 | 价值 | 难度 |
|----------|------|------|
| 龙虎榜明细 | 机构/游资买卖，短线关键信号 | 低（akshare已有） |
| 舆情/社交媒体 | 东财股吧热度、雪球讨论量，散户情绪 | 中 |
| 盘口数据（Level2） | 大单挂撤、委比委差，主力意图 | 高（付费） |
| 期权数据 | 隐含波动率变化，提前预判方向 | 中 |
| 可转债/股指期货 | 跨市场信号确认 | 中 |

### 11. 多日一致性追踪

- 连续3天被推荐的股票 vs 只出现1天的，前者更可靠
- 增加"连续推荐天数"字段到输出报告
- 首次出现时标注"新入选"，连续推荐时标注"持续看好(第N天)"

## P3：锦上添花

### 12. 交易维度补全

当前系统只解决了"选什么"，没有解决"怎么买"：

- **入场时机**：推荐的股票什么价位买？开盘买？回调到支撑位买？
- **出场策略**：止盈止损价位输出到报告（当前prompt里写了止损规则但没输出具体价位）
- **仓位分配**：10只推荐股，每只买多少？按 Borda 分数 / 波动率 / 等权？

### 13. 增量运行模式

- 当前每次全量运行（数据采集+全部专家分析），耗时长、成本高
- 增量模式：盘中只更新实时数据和资金流，复用当日已有的基本面分析
- 缓存不变数据：基本面/财务数据一天内不变，不需要每次重新让LLM分析

### 14. LLM 成本优化

- 链路A每次跑 4模型×7专家 = 28次LLM调用，token消耗大
- E1发送完整画像后E2-E6共享上下文（已实现），但可进一步精简prompt
- 考虑对"确信度低"的中间排名股票用cheaper model，只对Top候选用旗舰模型

### 15. 自动化调度

- 每日收盘后自动运行（定时任务）
- 盘前自动推送报告（邮件/微信/钉钉）
- 绩效周报自动生成

---

## 核心总结

> 当前系统擅长"选"，但缺少"验"和"管"——没有绩效验证就不知道选得准不准，没有仓位管理就无法把选股优势转化为实际收益。

优先级排序逻辑：
```
P0 绩效闭环  → 知道系统准不准（决策基础）
P1 质量提升  → 让系统更准（核心价值）
P2 持续增强  → 让系统越来越准（护城河）
P3 锦上添花  → 从选股到交易（完整闭环）
```
